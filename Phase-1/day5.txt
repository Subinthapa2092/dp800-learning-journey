DAY 5 — SEQUENCES
Date: 2026-04-05
Topic: SEQUENCE object, CYCLE vs NO CYCLE, 
       ALTER SEQUENCE, sp_rename

1. WHAT IS A SEQUENCE

→ Independent object that generates numbers automatically
→ NOT attached to any specific table (unlike IDENTITY)
→ Can be shared across multiple tables
→ You control when number is generated
→ Used for: TransactionID, InvoiceID, TicketID

DIFFERENCE FROM IDENTITY:
IDENTITY  → attached to ONE table only
            auto generates on INSERT only
SEQUENCE  → independent object
            can be used anywhere, anytime
            more flexible and powerful

2. CREATE SEQUENCE SYNTAX

CREATE SEQUENCE seq_transactionID
    START WITH 1
    INCREMENT BY 1
    MINVALUE 1
    MAXVALUE 9999999
    NO CYCLE;

-- Options explained:
START WITH  → first number generated
INCREMENT BY→ how much to add each time
MINVALUE    → lowest allowed value
MAXVALUE    → highest allowed value
NO CYCLE    → stops at MAXVALUE (throws error)
CYCLE       → restarts from MINVALUE after MAXVALUE

3. CYCLE vs NO CYCLE

CYCLE:
→ After reaching MAXVALUE goes back to MINVALUE
→ Keeps repeating forever like a circle
→ Example: slot numbers, queue positions
→ 1, 2, 3, 4, 5, 1, 2, 3... (if MAXVALUE=5)

NO CYCLE:
→ Stops permanently after reaching MAXVALUE
→ Throws error if you try to get next value
→ Example: TransactionID, InvoiceID, OrderID
→ Never repeat same ID = data integrity

NEPSE use:
TransactionID → NO CYCLE (never repeat)
SlotNumber    → CYCLE (reusable)

4. GET NEXT VALUE

-- Basic (shows "No column name")
SELECT NEXT VALUE FOR seq_transactionID;

-- Correct way (always add alias)
SELECT NEXT VALUE FOR seq_transactionID 
AS NextTransactionID;

-- Use in INSERT
INSERT INTO Transactions
    (CompanyId, BrokerCode, TradeDate, TradeType, Quantity, Price)
VALUES
(1, 'BR001', '2026-01-04', 'B', 100, 1465.00);
-- TransactionID auto generated by sequence

5. ALTER SEQUENCE (modify existing sequence)

-- Restart from beginning
ALTER SEQUENCE seq_transactionID
RESTART WITH 1;

-- Restart from specific number
ALTER SEQUENCE seq_transactionID
RESTART WITH 100;

-- Change increment
ALTER SEQUENCE seq_transactionID
INCREMENT BY 1;

-- Change maxvalue
ALTER SEQUENCE seq_transactionID
MAXVALUE 999999;

-- Change to CYCLE
ALTER SEQUENCE seq_transactionID
CYCLE;

-- Change to NO CYCLE
ALTER SEQUENCE seq_transactionID
NO CYCLE;

6. RENAME A SEQUENCE

-- Rename sequence
EXEC sp_rename 'seq_transactionID', 'SEQ_TransactionID';

-- Verify rename worked
SELECT name, current_value
FROM sys.sequences
WHERE name = 'SEQ_TransactionID';

⚠️ WARNING after renaming:
→ Tables using old sequence name in DEFAULT
   constraint will BREAK
→ Must drop and recreate DEFAULT constraint
   with new sequence name

7. DROP SEQUENCE

-- Delete permanently
DROP SEQUENCE seq_transactionID;

-- After dropping, this throws error:
SELECT NEXT VALUE FOR seq_transactionID;
-- Error: Cannot find object seq_transactionID

8. CHECK SEQUENCE STATUS

-- See all sequences and their details
SELECT
    name AS SequenceName,
    start_value,
    increment,
    current_value,
    minimum_value,
    maximum_value,
    is_cycling
FROM sys.sequences
WHERE name LIKE 'SEQ%';

9. TRANSACTIONS TABLE WITH SEQUENCE

CREATE TABLE Transactions (
    TransactionID   INT PRIMARY KEY
                    DEFAULT (NEXT VALUE FOR SEQ_TransactionID),
    CompanyId       INT NOT NULL,
    BrokerCode      VARCHAR(10) NOT NULL,
    TradeDate       DATE NOT NULL,
    TradeType       CHAR(1) NOT NULL,  -- B=Buy S=Sell
    Quantity        INT NOT NULL,
    Price           DECIMAL(10,2) NOT NULL,
    TotalAmount     AS (Quantity * Price),  -- computed column!
    FOREIGN KEY (CompanyId) REFERENCES Companies(CompanyID),
    CHECK (TradeType IN ('B','S')),
    CHECK (Quantity > 0),
    CHECK (Price > 0)
);

-- TotalAmount is COMPUTED column:
→ automatically calculated as Quantity x Price
→ you never insert it manually
→ SQL calculates it for you always

10. RENAME COLUMN AND TABLE

-- Rename column
EXEC sp_rename 'TableName.OldColumn', 'NewColumn', 'COLUMN';

-- Rename table
EXEC sp_rename 'OldTableName', 'NewTableName';

-- Rename database
ALTER DATABASE OldName MODIFY NAME = NewName;

-- Verify columns after rename
SELECT COLUMN_NAME, DATA_TYPE
FROM INFORMATION_SCHEMA.COLUMNS
WHERE TABLE_NAME = 'YourTable';

⚠️ WARNING after renaming columns:
→ Stored procedures using old name = BROKEN
→ Views using old name = BROKEN
→ Rename columns EARLY before creating
   stored procs and views

11. ERRORS LEARNED TODAY

Error 1: "No column name" in results
→ Forgot to add alias
→ Fix: SELECT NEXT VALUE FOR seq AS NextID

Error 2: Sequence showing 19, 21, 23...
→ Already consumed values 1-17 before
→ Fix: ALTER SEQUENCE RESTART WITH 1

Error 3: Sequence exhausted error
→ Hit MAXVALUE with NO CYCLE
→ Fix: RESTART or increase MAXVALUE or add CYCLE

12. REAL WORLD USAGE

SEQUENCE used in:
→ Banking: TransactionID, AccountID
→ NEPSE: TradeID, BrokerCode
→ Hospital: PatientID, AppointmentID
→ Invoice systems: InvoiceNumber

Why SEQUENCE over IDENTITY:
→ Multiple tables sharing one sequence
→ Reserve blocks of numbers in advance
→ More control over number generation
→ Can be used outside of INSERT statements

13. EXAM TIPS

Q: Difference between SEQUENCE and IDENTITY?
A: IDENTITY = one table, auto on INSERT
   SEQUENCE = independent, flexible, shared

Q: What happens when NO CYCLE hits MAXVALUE?
A: Throws error, sequence stops

Q: How to restart a sequence?
A: ALTER SEQUENCE name RESTART WITH n

Q: How to see current value of sequence?
A: SELECT current_value FROM sys.sequences

Q: What is a computed column?
A: Column automatically calculated by SQL
   TotalAmount AS (Quantity * Price)
   never inserted manually
